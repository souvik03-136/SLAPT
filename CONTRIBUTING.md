# Contributing to SLAPT

Thanks for wanting to make SLAPT even better! 🎵

## The Vibe Check

- **Be kind.** We're all learning.
- **Be clear.** Explain what and why.
- **Be patient.** Solo project = things take time.

## How to Contribute

### 🐛 Found a Bug?

1. Check if it's already reported in [Issues](https://github.com/souvik03-136/slapt/issues)
2. If not, open a new issue with:
   - What you tried to do
   - What happened
   - What you expected
   - Your SLAPT code (if relevant)

### 💡 Have an idea?

Open a [Discussion](https://github.com/souvik03-136/slapt/discussions) and let's talk about it!

### 🔧 Want to code?

1. Fork the repo
2. Create a branch: `git checkout -b feature/your-cool-thing`
3. Make your changes
4. Test it (make sure it slaps)
5. Commit: `git commit -m "add: your cool thing"`
6. Push: `git push origin feature/your-cool-thing`
7. Open a Pull Request

**Commit message style:**
- `add:` new features
- `fix:` bug fixes
- `docs:` documentation changes
- `refactor:` code improvements
- `test:` adding tests

### 🎵 Want to add a genre template?

Hell yeah! Open an issue or PR with:
- Genre name
- Typical BPM range
- Common patterns
- Characteristic effects
- Example code

### 📝 Improve docs?

Docs are never perfect. If something's confusing, make it clearer!

## Code Style

- Keep it readable (like the rest of SLAPT)
- Comment when necessary
- Write tests if you can
- Make sure examples work


---

**Remember:** Every contribution counts. Documentation fixes, typo corrections, feature suggestions—it all helps make SLAPT better for everyone.

Let's make music production accessible to everyone. 🚀
